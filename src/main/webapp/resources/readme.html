Fronsetia - Read Me

This is the README file for Fronsetia (Free Online Service Testing Application), a web application that allows testing webservices.

Author: Bogdan Drozdowski, bogdro %AT% users . sourceforge . net

License: AGPLv3+.

The project's website is available at fronsetia.sourceforge.io.

Requirements for compiling the application:

  • the development kit for the Java language (JDK),
  • Apache Maven.

Type mvn package to compile the application. Alternatively, you can use a make utility and run just make (which calls Maven now anyway).

By default, a self-contained package, with all the required libraries inside, is built.

You can also build a minimal package without all the required libraries inside by adding 

	<scope>provided</scope>

tags to each <dependency> in pom.xml (expect where already present and has a different value) and re-building the application (running mvn package).

How to run:

  • If you've built a self-contained package with all the required libraries inside, simply deploy the application on your server.

  • If you've built a minimal package without all the required libraries inside:

    1. Add the JAR files listed in pom.xml in the versions given there (and their dependencies) to your server's CLASSPATH.
    2. Re-start your server if necessary.
    3. Deploy the application on your server.

You can get all the JAR libraries from Maven Central (as for all Maven libraries) or the respective projects' pages:

  • wsdl4j.jar - the WSDL4J project,
  • xbean.jar - Apache XmlBeans,
  • httpclient.jar - Apache HttpComponents,
  • httpcore.jar - Apache HttpComponents,
  • commons-logging.jar - Apache HttpComponents,
  • commons-codec.jar - Apache Commons Codec,
  • axiom.jar - Apache Axiom,
  • axis2-saaj.jar - Apache Axis2.

The jakarta.jakartaee-api dependency should be provided by your JavaEE servlet container/server.

The junit.jar dependency (and all other with <scope>test</scope>) are not required to build or run the application.

How to use:

  1. Access the application on the server from a browser. The exact URL depends on your server and the name you gave the application.
  2. Type the WSDL address on the application's main page and proceed.
  3. Choose the operation to run, fill the data and press the button with the operation's name.

You can start from the page with the list of operations by passing the address of the WSDL file in the URL.
